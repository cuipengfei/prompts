# Jichang

在公开资料极不干净的场景下评估机场 / jichang / proxy subscription provider。目标不是替用户“推荐机场”，而是把候选发现、证据可信度、广告污染、近期风险和低暴露测试优先级拆开，避免把 SEO 榜单、优惠码软文或一次测速写成可靠结论。

## 适用范围

当用户问以下内容时使用本技能：

- “这个机场靠谱吗？”、“有没有稳定机场？”、“jichang options”
- 某个机场、登录页、订阅服务、VPN/proxy provider 的可信度
- DuyaoSS、茶波、机场评测站、Telegram 口碑、GitHub 榜单、优惠码页面是否可信
- 在审查/监管收紧、服务倒闭、线路被拔、支付或域名受影响背景下筛选候选

允许做公开来源尽调：域名关系、公告、付款风险、隐私/账号暴露风险、近期负面和测速污染。不要把本技能用于搭建代理、绕过检测、隐藏流量特征、生成规避审查技术步骤，或提供保证可用的名单。

## 核心原则

- 不给稳定承诺：只能说“公开证据显示更适合/不适合小额测试”，不能说“一定稳定”“可靠主力”。
- 候选发现不等于背书：SEO 榜单、优惠码站、GitHub 排名、YouTuber/KOL 可用于发现名字；正向宣传默认不加分。
- 弱正向证据可以用，但要降级：长期、多日期、多运营商、有失败记录的观察源可支持“小额测试优先级”，不能支持“可靠”。
- 近期测速要降权：测速只证明某时间、某线路、某运营商、某测试环境下出现过表现；不能证明用户本地长期可用。
- 负面信息优先：断流、客服失联、官网打不开、TG 禁言、频繁换域名/订阅、终身套餐、大促销都要显式标红。
- 暴露控制是底线：若用户已自行决定继续验证，只说明如何限制风险暴露：试用优先、不年付、不充值余额、分时段记录失败。

## 来源分层

| 层级 | 用途 | 处理方式 |
|------|------|----------|
| A：官方/一手资料 | 公告、套餐、服务条款、状态页、客服公告 | 可确认“它自己怎么说”，仍不能自证稳定。 |
| B：长期测速/观察源 | DuyaoSS、持续多月有日期/运营商/时段的测速页 | 可证明“近期被观察到”，不能单独证明可靠。 |
| C：有负面记录的评测博客 | 茶波、润土、爱机场等会写不建议新购/断流/带宽不足的来源 | 可用于识别风险和动态，仍可能有推广关系。 |
| D：社区口碑 | Telegram、论坛、评论、社媒 | 看负面模式和时间集中度，不把单条评论当事实。 |
| E：SEO/affiliate 榜单 | 十大机场、优惠码、直达官网、最高折扣、闭眼入 | 只用于发现候选；正向宣传不加分。 |

## 执行流程

1. **定义用户真实需求**
   - 先判断用户最怕什么：断线、慢、AI 不可用、流媒体不可用、远程办公不稳、游戏延迟，还是跑路。
   - 若用户只给名字或 URL，默认目标是“判断是否值得小额测试”。

2. **候选发现**
   - 搜索候选名称、别名、域名、登录页标题、官网域名、Telegram 群名。
   - 记录是否存在同名混淆或多个域名体系；无法确认关系时标为“域名关系未验证”。

3. **证据采集**
   - 至少尝试覆盖：官方/登录页、近期测速、负面评测或社区口碑、SEO/affiliate 页面。
   - 对每条证据记录日期、来源类型、它能证明什么、不能证明什么。

4. **广告污染扣分**
   - 出现“永久免费”“永不被墙”“稳如泰山”“闭眼入”“传家宝”“行业第一”“最高折扣”“限时优惠”“点击直达官网”等词，标记为广告污染。
   - 大量优惠码、只有优点无缺点、无日期测速图、无运营商/地区/时段说明，也标记为广告污染。

5. **风险分层**
   - 输出短标签，不输出 Top 10 排名：
     - **可小额试**：近期证据较多，负面可识别，付款和账号暴露可控；仍不代表稳定。
     - **观察**：有部分证据，但单源、推广味重或近期动态不足。
     - **跳过**：近期负面明显、付款风险高、只靠 SEO/affiliate、终身/年付强推，或客服/公告不透明。
     - **冲突**：来源之间互相打架，无法判断哪边更新或更可信。
     - **身份未确认**：域名、登录页、TG 群、品牌名关系没对上。

6. **低暴露尽调计划**
   - 若用户已自行决定测试，只说明如何降低损失面：优先试用或最低暴露方式；不年付，不买终身，不充值余额。
   - 至少测 3 天：工作日白天、工作日晚高峰 20:00-23:00、周末晚高峰。
   - 记录连接成功率、晚高峰可用节点比例、AI/网页/视频失败次数、断流次数、客服响应和公告透明度。

## 输出格式

```markdown
## 结论
- <候选>：<可小额试 / 观察 / 跳过 / 冲突 / 身份未确认>
- 这不是稳定性保证，也不是购买建议；只是基于公开证据的低暴露测试优先级。

## 证据
- 来源：<URL/站点/频道/页面名>
  - 类型：<官方/长期测速/负面评测/社区口碑/SEO-affiliate>
  - 新鲜度：<日期/最近更新时间/未知>
  - 支撑：<它能证明什么>
  - 不支撑：<它不能证明什么>
  - 偏见/污染：<推广、优惠码、单次测速、无失败记录、身份不明等>

## 风险标记
- 广告污染：<有/无/未知>，证据：...
- 域名关系：<已验证/未验证/混乱>，证据：...
- 近期负面：<有/无/未知>，证据：...
- 付款风险：<试用/月付/只年付/未知>

## 建议动作
1. <若用户已自行决定测试：只用试用或最低暴露方式，不预存余额，不年付，并记录测试窗口与失败指标>
2. <若不宜继续：说明为什么先跳过或等待更多证据>
```

## 禁止事项

- 不要输出“稳定机场推荐榜”“保证可用”“永不被墙”“跑路风险低所以可年付”。
- 不要输出“应该买”“值得买”“推荐购买”“首选机场”等购买导向结论；可以给公开证据下的低暴露测试优先级。
- 不要把 affiliate/SEO 页面中的正向话术当成事实。
- 不要把一次测速、单一运营商测速、单个评测博客直接升级成可靠结论。
- 不要提供规避审查、绕过封锁、隐藏流量特征的操作教程。
- 不要推荐用户年付、终身套餐或大额充值。
